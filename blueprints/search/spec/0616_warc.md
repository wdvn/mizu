# 0616 — WARC Package + cc warc Commands

## Overview
pkg/warc provides streaming read/write access to WARC 1.1 files
with automatic gzip detection and a DuckDB import bridge.

## WARC File Format (relevant subset)
A .warc file is a sequence of records:
  WARC/1.1\r\n
  Key: Value\r\n
  ...\r\n
  \r\n
  <body: Content-Length bytes>
  \r\n\r\n

A .warc.gz file is a concatenated gzip stream where each
gzip member decompresses to exactly ONE WARC record (CC convention).
Magic bytes: 0x1f 0x8b (standard gzip).

## pkg/warc API

### Reader
  r := warc.NewReader(f)   // auto-detects gzip
  for r.Next() {
      rec := r.Record()    // *warc.Record
      io.Copy(io.Discard, rec.Body)  // must drain before Next()
  }
  if err := r.Err(); err != nil { ... }

### Writer
  w := warc.NewWriter(f)
  w.WriteRecord(rec)
  w.Close()

### Importer (Reader → DuckDB)
  db, _ := warc.OpenRecordDB(dir)
  im := warc.NewImporter(db, warc.ImportOptions{MIMETypes: []string{"text/html"}})
  im.Import(ctx, r, func(s warc.ImportStats) { fmt.Println(s) })
  db.Close()

## CLI: search cc warc

  cc warc get      --file X --offset Y --length Z   (existing: fetch one record)
  cc warc list     [--crawl X] [--limit N] [--json]
  cc warc download [--crawl X] [--file N] [--workers 4]
  cc warc extract  [--crawl X] [--file N] [--mime text/html] [--status 200] [--out ndjson]
  cc warc import   [--crawl X] [--file N] [--mime text/html] [--status 200] [--workers 4]
  cc warc query    [--crawl X] [--domain X] [--limit 20] [--out table|json]

## Storage Layout
  $HOME/data/common-crawl/{crawl-id}/
    warc/           # downloaded .warc.gz files
    warc-import/    # 8-shard DuckDB (warc_000.duckdb ... warc_007.duckdb)

## DuckDB Schema (warc-import)
  CREATE TABLE IF NOT EXISTS records (
    warc_file    VARCHAR,
    record_id    VARCHAR,
    record_type  VARCHAR,
    url          VARCHAR,
    domain       VARCHAR,
    crawled_at   TIMESTAMP,
    http_status  INTEGER,
    mime_type    VARCHAR,
    language     VARCHAR,
    title        VARCHAR,
    description  VARCHAR,
    body         BLOB,
    body_length  BIGINT,
    http_headers VARCHAR   -- JSON
  )

## Bubbletea Progress Model
Long-running commands (download, import, extract, fetch, index, parquet)
use a reusable cli/ui.ProgressModel with:
  - Per-item progress bar (bubbles/progress)
  - Overall progress bar
  - Speed + ETA line
  - Final summary on done

The model is driven by tea.Program.Send() from worker goroutines.
